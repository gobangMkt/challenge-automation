# 블로그 챌린지 앱 — 설계 스펙

> 단일 진실 소스. raw 수집자료는 `context.md` 참조.
> 확정: 2026-06-12 (brainstorming 완료) · 스택/도메인/슬라이스 fix

## 1. 개요
- **무엇**: 블로그 챌린지("취준 블로그 마스터즈 / 방학스펙업")의 신청·주차운영·정산을 한 곳에서 처리하는 웹앱.
- **누구**: ①참가자(신청·주차제출) ②운영자(챌린지 생성·선발·진행·정산).
- **왜**: 현재 구글폼 4종 + 노션 수작업으로 흩어진 운영을, 앱으로 통합하고 매주 진행을 **스케줄 자동화**(회차오픈·알림톡·리마인드)한다.

## 2. 핵심 결정 (fix)
| 항목 | 값 |
|---|---|
| 스택 | GitHub Pages(정적 SPA) + Google Apps Script(API·트리거) + Google Sheets(DB) + SOLAPI(알림톡) + Notion API(아티클 read) |
| 참가자 식별 | 로그인 없음. **휴대폰번호 = 키** (모든 폼 공통, 리워드 수령) |
| 다중 챌린지 | 운영자가 여러 개 생성·운영 |
| 자동화 | GAS 시간트리거 매일 1회: 회차 자동오픈+알림톡, 미제출 리마인드 |
| 노션 | 결과는 노션에 **안 씀**. 아티클만 노션 「작성기록」에서 **read** |
| 회차 아티클 | 운영자 수동 배정 + 비우면 Date 최신순 **자동 fallback** |

## 3. 도메인 모델 & 데이터 (Google Sheets 탭)

### Challenges
| 컬럼 | 설명 |
|---|---|
| challengeId | 고유 ID (슬러그) |
| name | 챌린지명 |
| status | 모집중 / 선발중 / 진행중 / 종료 |
| 모집시작, 모집마감, 발표일 | 신청·선발 일정 |
| 시작일 | 1회차 시작 |
| 총회차 | 기본 10 |
| 오픈요일 | 회차 오픈 요일 (예: 화) |
| 마감요일/마감오프셋 | 회차 마감 기준 |
| rewardPerPost | 제출 1건당 활동비 |
| excellentMultiplier | 우수활동자 배수 (기본 2) |
| notionDbId, notionFilterType | 아티클 소스(가변), 필터값("혼잘주거") |
| operatorToken | 운영자 대시보드 접근 토큰 |
| openchatUrl | 문의 오픈카톡 링크 |

### WeekMissions
challengeId, 회차(1~N), 미션제목, 미션본문, **articleName, articleUrl**(노션 배정 결과 캐시), 오픈일, 마감일, 상태(대기/오픈/마감)

### Participants
challengeId, **phone(키)**, name, blogUrl(참가 블로그 1개), 신청일시, 동의(Y), **status(신청/선발/탈락/완주/중도이탈)**, 비고

### Submissions
challengeId, phone, 회차, postUrl(그 주 게시물), 제출일시, (검수상태)

### Wrapup
challengeId, phone, blogUrl, 작성갯수(0~N), 우수활동자(Y/N), 제출일시

### NotifyLog
challengeId, 회차, type(오픈/리마인드/선발/완주), phone, 발송시각, 결과(성공/실패+사유)

## 4. 화면 & 기능

### 참가자 (공개, `?c=<challengeId>`)
1. **랜딩+신청**: 챌린지 소개(카피·일정·리워드·오픈카톡) + 신청폼(성함/휴대폰/참가블로그URL/동의). 모집마감 후 "마감" 상태.
2. **주차 제출**: 휴대폰 입력 → 본인 식별 → **현재 오픈 회차 자동판별** → 그 주 게시물 URL 제출. 화면에 **이번주 미션+참고 아티클** 노출 + **본인 진행현황(N/총회차) 회차별 제출 뱃지**.
3. **마무리**: 작성갯수·우수활동자 등 제출.

### 운영자 (대시보드, operatorToken 보호)
1. **챌린지 생성·설정**: 위 Challenges 필드 + 노션 DB소스/필터.
2. **회차 미션 편집**: 10회차 미션제목·본문 입력, **아티클 배정**(노션 혼잘주거 목록에서 선택 / 비우면 최신순 자동).
3. **선발 처리**: 신청자 목록 → 선발/탈락 토글 → 선발자에게 "참가확정" 알림톡 발송.
4. **제출현황 매트릭스**: 참가자 × 회차 그리드, 제출/미제출, 완주율.
5. **알림 로그 + 수동 발송**: 발송 이력, 실패 재발송, 임의 회차 수동 발송.
6. **결과·정산**: 참가자별 제출수 → 활동비 계산(우수 2배), 정산표 export(CSV).

## 5. 자동화 (GAS 시간트리거, 매일 1회)
- **회차 오픈**: 오늘이 진행중 챌린지의 회차 오픈일 → 해당 WeekMission 상태=오픈, 아티클 자동fallback 확정, 선발 참가자 전원 **알림톡**(미션+아티클+제출링크).
- **리마인드**: 회차 마감 D-1 → 그 회차 **미제출 참가자에게만** 리마인드 알림톡.
- **회차 마감**: 마감일 경과 → 상태=마감, 다음 회차로.
- **종료**: 마지막 회차 마감 → 챌린지 status=종료, 마무리폼 안내 알림톡.

## 6. 노션 아티클 연동 (read-only)
- GAS가 Notion API로 `notionDbId` 데이터소스 쿼리, 필터 `type contains notionFilterType("혼잘주거")`.
- 사용 필드: Name(제목), URL(링크), Date. Date 내림차순.
- 회차 배정: 운영자 수동 선택 우선, 미배정 회차는 오픈 시 최신순으로 자동 채움(WeekMissions에 캐시).
- 인증: Notion Integration 토큰(read), 대상 DB를 인티그레이션에 공유.

## 7. 아키텍처 / API
- **프론트**(Pages): 정적 HTML/JS SPA. 참가자 뷰 + 운영자 뷰 라우팅. fetch로 GAS 호출.
- **백엔드**(GAS Web App): `doGet`(조회)·`doPost`(쓰기) 단일 엔드포인트, action 파라미터로 분기. 시트 R/W, SOLAPI·Notion 호출, 시간트리거 핸들러.
- **시크릿**(Script Properties): SOLAPI 키/발신/템플릿ID, Notion 토큰, operatorToken 해시.

## 8. 비기능 / 제약
- **개인정보**: 성명·휴대폰 수집, 이벤트 종료 시까지 보유(동의문구). 시트 접근 최소화, 토큰 보호.
- **알림톡**: SOLAPI 채널·템플릿 사전 승인 필요(오픈/리마인드/선발/완주 템플릿).
- **인증**: 운영자 토큰 기반(기존 앱 패턴). 참가자 무인증.
- **CORS**: GAS Web App 배포 권한 = "모든 사용자", Pages에서 fetch.

## 9. 개발 순서 (수직 슬라이스)
1. **신청** — 랜딩+신청폼 → 시트저장 → 운영자 명단 조회 (엔드투엔드 첫 조각)
2. **챌린지 생성·설정** — 운영자가 챌린지/일정/회차 골격 생성
3. **선발** — 신청자 선발/탈락 + 선발 알림톡
4. **주차 제출** — 회차 자동판별 제출 + 본인 진행현황 + 미션/아티클 노출
5. **운영 대시보드** — 제출현황 매트릭스
6. **스케줄 자동화 + SOLAPI** — 회차오픈·리마인드 알림톡, 노션 아티클 read
7. **마무리 + 정산** — 마무리폼 + 활동비 정산표

## 10. 가정 / 미해결
- 회차당 아티클 1개 기본(여러 개 허용은 추후). 
- 우수활동자 판정: 마무리폼 자기신고(Y/N) 기준 — 운영자 검수로 보정 가능(추후).
- 게시물 URL 진위 검수는 수동(자동 크롤 검증은 범위 외).
- 디자인 테마: 2단계 @ui-ux-pro-max에서 재설계(포스터 톤은 참고만).
