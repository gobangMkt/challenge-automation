- 컨셉: 취준생이 SEO 글쓰기·실무 감각을 쌓고 포트폴리오 완성. 검색 키워드 활용 블로그 콘텐츠를 10주간 작성.

- 목표 지표 예: 참가자 수/비율, 블로그 수(인당), 사용금액(개당 단가)
- DB 속성: Name, Number, 목표, 방법, 변수, 진행기간, 상태(준비/🚨진행중/완료), 담당자, 임펙트(★~★★★★★), 결과
- 배포 단계에서 "유저 설문 필요 시 신청 폼에 설문 항목 추가" 패턴 존재.

## 4. 구글폼 4종 실제 항목 (원문)

### 신청 폼 — "🔥10주 완성, 취준 블로그 마스터즈"
상단설명: 취준생이 실무경험·포트폴리오 완성하는 10주 챌린지 소개
1. 성함 (단답, 필수) — 예시) 김고방
2. 연락받으실 휴대폰 번호 (단답, 필수) — 000-0000-0000
3. 챌린지에 참가할 블로그 URL (단답, 필수) — 본인 명의 블로그 1개만 / 도배·어뷰징 불가
4. 개인정보 수집·이용 동의 (체크박스, 필수)

### 1주차 폼 — "[1주차] 취준 블로그 마스터즈"
상단설명: SEO 글쓰기·실무 감각, 10주간 검색 키워드 활용 콘텐츠 작성
1. 성함 (단답, 필수) — 공백 없이
2. 연락받으실 휴대폰 번호 (단답, 필수) — 000-0000-0000
3. 이번주에 작성한 블로그 URL (단답, 필수) — 본인 블로그가 아닌 **게시물** URL
4. 개인정보 수집·이용 동의 (체크박스, 필수)

### 운영 폼 (2~10주차 공용) — "[2주차] 취준 블로그 마스터즈"
상단설명(회차별로 달라짐): 2주차 = "아티클 참고하되 그대로 쓰지 말고 1,000자 이상 자연스러운 SEO 콘텐츠로 작성"
1. ✏️ 성함 (단답, 필수)
2. ✏️ 연락받으실 휴대폰 번호 (단답, 필수)
3. ✏️ 이번주에 작성한 블로그 URL (단답, 필수) — 게시물 URL
4. 개인정보 수집·이용 동의 (체크박스, 필수) — 성명·전화번호, 이벤트 종료 시까지 보유

### 마무리 폼 — "🎉챌린지 완주를 축하합니다 🎉"
상단설명: 10주 완성 블로그 캠퍼스 프로젝트 완료 축하 + 리워드 안내
1. ✏️ 성함 (단답, 필수)
2. ✏️ 리워드를 수령할 휴대폰 번호 (단답, 필수)
3. ✏️ 챌린지에 참가한 블로그 URL (단답, 필수) — 본인 명의 1개 / 도배·어뷰징 불가
4. ✏️ 작성한 블로그 갯수 (드롭다운, 필수) — 0개~10개, "정확히 입력 / 우수활동자는 리워드 2배"
5. ✏️ 우수활동자 여부 (객관식, 필수) — Y / N
6. 개인정보 수집·이용 동의 (체크박스, 필수)

## 5. 핵심 도메인 규칙 (폼에서 확정)
- **식별키 = 휴대폰번호** (모든 폼 공통, 리워드 수령에도 사용).
- 신청 시 **참가 블로그(채널) 1개 등록** → 매주 **그 주 게시물 URL** 제출.
- **회차별 '이번주 미션' 안내문이 회차마다 다름** (폼 상단설명). 운영자가 10회차 미션을 미리 작성. → 제출화면 상단 + 알림톡 본문에 삽입.
- 마무리 정산: **작성갯수 기반**, **우수활동자 = 리워드 2배**.
- 동의 문구: 성명·전화번호 수집, 이벤트 종료 시까지 보유.

## 7. 설계안 (확정 직전)

### 아키텍처
- [GitHub Pages] 정적 SPA — 참가자 화면 + 운영자 대시보드
- [GAS] doGet/doPost API — 검증·로직·SOLAPI 호출, 시간트리거
- [Google Sheets] DB
- [SOLAPI] 카카오 알림톡

### 데이터 모델 (시트 탭)
- **Challenges**: 챌린지ID, 이름, 시작일, 총회차(10), 주차_오픈요일, 주차_마감요일, 상태, 리워드정책, 운영자토큰
- **WeekMissions**: 챌린지ID, 회차(1~10), 미션제목, 미션본문 (← 회차별 안내문)
- **Participants**: 챌린지ID, 휴대폰(키), 성함, 참가블로그URL, 신청일시, 동의, 상태
- **Submissions**: 챌린지ID, 휴대폰, 회차, 게시물URL, 제출일시
- **Wrapup**: 챌린지ID, 휴대폰, 참가블로그URL, 작성갯수, 우수활동자Y/N, 제출일시
- **NotifyLog**: 챌린지ID, 회차, 타입(오픈/리마인드), 휴대폰, 발송시각, 결과

### 화면
- 참가자(공개, 챌린지 링크): 신청 / 제출(휴대폰→현재회차 자동판별→게시물URL, 본인 진행현황 N/10) / 마무리
- 운영자(토큰 보호): 챌린지 생성·설정(+10회차 미션 입력) / 참가자 명단 / 제출현황 매트릭스(참가자×10회차) / 알림 발송로그·수동발송 / 결과집계·리워드 정산

### 자동화 (GAS 시간트리거 매일 1회)
- 오픈요일 → 회차 오픈 + 전원 알림톡(이번주 미션+제출링크)
- 마감 D-1 → 미제출자에게만 리마인드 알림톡
- 마감일 → 회차 마감 후 다음 회차

### 개발 순서 (수직 슬라이스)
1. 신청 (참가자 신청폼 → 시트저장 → 운영자 명단) — 엔드투엔드 첫 조각
2. 운영자 챌린지 생성/설정 (+ 10회차 미션)
3. 주차별 제출 + 본인 진행현황
4. 운영자 제출현황 대시보드
5. 스케줄 자동 알림톡(오픈/리마인드) + SOLAPI
6. 마무리 + 결과집계/리워드 정산
